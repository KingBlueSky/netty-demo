1. 三种分布式事务实现方式比较

   |                      | XA的二阶段提交 | 三阶段提交 | 分布式消息的最终一致性 |
   | :------------------: | :------------: | :--------: | :--------------------: |
   |    算法一致性类别    |    强一致性    |  强一致性  |       最终一致性       |
   |       执行方式       |    同步执行    |  同步执行  |        异步执行        |
   |     同步阻塞问题     |       有       |     无     |           无           |
   |     单点故障问题     |       有       |     无     |           无           |
   |    数据一致性问题    |      存在      |    存在    |      最终会不存在      |
   |      系统并发度      |       低       |     中     |           高           |
   | 分布式事务系统的性能 |       低       |     中     |           高           |

   

2. 刚性事务与柔性事务

   1. 刚性事务：遵循ACID原则，具有强一致性

   2. 柔性事务：在业务规则下，容忍一定时间内的数据不一致，但会最终达到数据的一致性
